## 🔐 Authentication & Security

The API uses **JWT (JSON Web Tokens)** for authentication.

### Core Endpoints:
1. **Login:** `POST /api/auth/login`
   - Body: `{ "username": "admin@admin.com", "password": "password123" }`
   - Returns: `accessToken` and `refreshToken`

2. **Test Expiration Login:** `POST /api/auth/login-short-token?expiryMinutes=1`
   - **Agent Tip:** Use this endpoint during development to quickly test your Axios interceptors and token refresh logic without waiting 60 minutes!

3. **Refresh Token:** `POST /api/auth/refresh-token`
   - Body: `{ "refreshToken": "YOUR_REFRESH_TOKEN" }`

### Default Admin Credentials:
- **Email/Username:** `admin@admin.com`
- **Password:** `Admin123!`

### Agent Instructions for Auth:
> **For the AI Agent:** Implement an Axios HTTP interceptor to catch `401 Unauthorized` responses. When a 401 occurs, automatically pause pending requests, hit the `/refresh-token` endpoint with the stored refresh token, update the local storage with the new tokens, and replay the paused requests.

## 📡 Real-Time Notifications (SignalR)

The backend pushes real-time events to the frontend via SignalR. 
- **Hub Endpoint:** `/realtimeHub`
- **Events to Listen For:**
  - `ReceiveNotification`: Triggered when an employee requests a leave, or a manager approves/rejects a request.
  - `ReceiveMessage`: General system messages.
  - `BonusStatusUpdated`: Triggered when a bonus request changes status.

## 🏗️ API Architecture & Standards

- **JSON Naming:** All API responses and request bodies use strict `camelCase`.
- **Date Formatting:** Dates are returned in ISO 8601 format (e.g., `2026-07-20T14:30:00Z`).
- **Authorization Header:** All protected endpoints require the header: `Authorization: Bearer <Your_Access_Token>`
- **Response Wrapping:** Most listing endpoints return arrays directly, but paginated endpoints may return an object containing `data`, `pageNumber`, `pageSize`, and `totalRecords`.

Here is a high-level overview of the main controllers and their purpose:

1. **Auth (`/api/auth`)**: Login, registration, OTP, and token refresh.
2. **Leave Requests (`/api/leave-requests`)**: Applying for time off, retrieving balances, and manager approvals.

5. **Payroll (`/api/payroll`)**: Uploading Excel salary sheets and distributing PDF slips via email.
6. **Bonuses (`/api/bonuses`)**: Requesting and approving financial bonuses or penalties.
7. **Departments & Sections (`/api/departments`, `/api/sections`)**: Organizational structure.
8. **Settings (`/api/settings`)**: Global company work settings.

## 🤖 Directives for the Frontend AI Agent

1. **Always Consult Swagger First:** The Swagger UI (`/swagger/index.html`) is your source of truth. Every endpoint has XML descriptions (`<summary>`) explaining its exact behavior and required payload.
2. **Handle Enum Strings Correctly:** The backend converts Enums to Strings in JSON (e.g., `"Pending"`, `"Approved"`, `"Rejected"`). Ensure your frontend types/interfaces use String Unions rather than numbers.
3. **Multipart/Form-Data:** Endpoints that accept files (like uploading an attachment for a Leave Request or uploading an Excel Salary Sheet) require `Content-Type: multipart/form-data`.
4. **Graceful Error Handling:** If the API returns a `400 Bad Request` or `500 Internal Server Error`, it typically includes a JSON body with a `message` property (e.g., `{ "message": "Reason for failure" }`). Display this `message` directly in a Toast or Snackbar to the user.
